1. Background and Objectives

Starting from a 300× deep-sequencing BAM derived from the reference cell line NA12878, artificial cfDNA samples were constructed at 10% simulated fetal fraction (FF) by injecting FF-informative SNP reads. Two variant panels were independently spiked in: 28 hotspot pathogenic SNVs across 21 genes, and 104 dominant/X-linked pathogenic SNVs across 104 genes.

Before error filtering, the pipeline reported 15,117 target variants — far too many for reliable clinical interpretation at the expected fetal VAF of ~5.5% (= FF/2). This report documents the layered error filtering approach implemented to suppress false positives while preserving all spiked-in true positives.

2. Error Filtering Architecture

Three complementary layers were implemented. Each layer targets a distinct error source and operates at a different point in the pipeline.
#LayerLocationMechanismTarget Error Type
L1 Strand-bias bcftools filter FILTER_VARIANTS (Nextflow) FORMAT/ADF[0:1] ≥ 1 AND FORMAT/ADR[0:1] ≥ 1 Single-strand sequencing errors (PCR/chemistry artefacts)
L2 Extreme strand-ratio filter variant_analysis.py Forward/reverse ALT ratio ≤ 20× One-sided PCR amplification bias, homopolymer artefacts

5. Residual False Positive Analysis (20 FP)

After all three filtering layers, 20 non-spiked variants remain classified as fetal_specific. These are identical in both panels (same baseline BAM). Two distinct sub-types are observed.

FP Type A — Read Duplication Artefact (simulation-specific)

Alt-count clustering at exact multiples of 6 and 12 is the diagnostic signature. The baseline BAM was constructed by tripling all reads (100× → 300×). Any sequencing error present in the original 100× data is amplified exactly 3× (or 6× if 2 copies originally existed), producing alt counts of 6 or 12 at depths of ~200–300×. This pattern does not occur in real clinical cfDNA where read counts are Poisson-distributed.

FP Type B — Homopolymer Indel Artefact (real-world concern)

Homopolymer run slippage errors are a known sequencing artefact in all short-read platforms. In regions with ≥8 consecutive identical bases (e.g., poly-T runs), the polymerase can stutter, producing 1-bp deletion artefacts. Unlike the duplication pattern above, these can occur in real clinical samples and should be addressed.

6. Fetal Fraction Confidence Interval Correction

FF estimation uncertainty is now propagated into the noise thresholds. When FF is estimated with wide confidence intervals (unreliable), noise filtering thresholds are automatically raised to compensate.

📐

Effective FF for noise thresholds

effective_ff_noise = min(FF + z × SE, CI_upper) where SE = (CI_upper − CI_lower) / (2 × 1.96), z = 1.0

This raises the noise_floor and sub-fetal noise boundary when FF is uncertain, making classification more conservative.

🎯

Fetal window centre — always point estimate

The fetal window is centred on FF/2 (not effective_ff_noise) to preserve sensitivity for true fetal variants.

⚠️

Wide CI → confidence downgrade

When relative CI width > 15%, fetal_specific variants are assigned confidence: medium instead of high, signalling to the clinician that FF estimation is less reliable.

7. Ambiguous Variants (43)

43 variants remain classified as ambiguous after all filters. Their VAF distribution (min 6.9%, median 14.3%, max 84.6%) places them in the intermediate range between the fetal window upper bound (6.85%) and the maternal heterozygous range (40–60%). These likely represent:

  • NA12878-specific low-frequency somatic or mosaic variants
  • Allele-specific sequencing depth biases at certain capture probe boundaries
  • Genuine maternal variants at lower-than-typical heterozygous frequency
These are not clinically actionable in the current framework and do not represent false-positive fetal-specific calls. The count (43) is stable across both variant panels, confirming they originate exclusively from the baseline BAM noise profile rather than the spiked-in variants.

8. Implementation Status and Next Steps

Done

L1 — Strand-bias bcftools filter (ADF/ADR ≥ 1)

Removes 91% of raw variant calls that carry ALT reads on only one strand. Applied after multiallelic normalisation so per-allele strand counts are accurate.

Done

L2 — Extreme strand-ratio filter (≤ 20×)

Catches high-VAF variants with extreme forward/reverse imbalance that pass L1 (both strands carry ≥1 read). Catches homopolymer and PCR bias artefacts.

Done

L3 — FF-adaptive noise classification

Three-zone classification: noise_floor (FF/4), sub-fetal noise ([FF/4, FF×0.375]), fetal window (FF/2 ± 25%). Tighter tolerance reduces FP fetal_specific calls by 96.7% vs. baseline.

Done

FF CI-based correction

Noise thresholds raised by z×SE when CI is available. Wide CI triggers confidence downgrade. Effective FF and CI info stored in result.json for traceability.

Next

Homopolymer indel filter

Flag or filter 1-bp indels in runs of ≥ 8 identical bases. Can be applied in bcftools with a BED mask of known homopolymer regions, or post-hoc in variant_analysis.py by inspecting REF length and sequence context.

Next

Panel of Normals (PoN)

Build a background VCF from multiple baseline runs (NA12878 and blank controls). Any position recurrently noisy across ≥ 2 normals is added to a blocklist and suppressed in all samples.

Future

UMI-based consensus calling

Requires library protocol change to include UMI adapters. Enables read-family deduplication and error correction at the source, eliminating the root cause of PCR and chemistry errors rather than filtering downstream.
