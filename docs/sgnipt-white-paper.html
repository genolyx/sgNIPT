Abstract

Single-Gene Non-Invasive Prenatal Testing (sgNIPT) identifies fetal pathogenic variants from cell-free DNA (cfDNA) in maternal plasma. Because fetal DNA represents only a small fraction of the total cfDNA pool (the fetal fraction, FF), variant allele frequencies (VAF) of fetal-specific variants are predictably low — approximately FF/2. This white paper describes the sgNIPT pipeline's fetal fraction estimation algorithm, variant origin classification logic, and the in silico simulation framework used to generate validation samples FF10_sgNIPT_hotspot and FF10_sgNIPT_dominant_xlinked.

1. Biological Model of cfDNA in NIPT

Maternal plasma cfDNA is a mixture of maternal-derived and placenta-derived (fetal) DNA fragments. At any genomic locus, the observed VAF reflects both contributions:

Observed VAF = (maternal contribution × maternal AF) + (FF × fetal AF)

For a locus where the mother is homozygous reference (hom-ref) and the fetus carries a heterozygous pathogenic allele (the canonical "paternal inheritance" scenario), the fetal allele is present in FF/2 of all cfDNA molecules:

Observed VAF ≈ FF / 2 (e.g. FF = 10% → VAF ≈ 5%)

For maternal heterozygous variants, the allele is present at ~50% in maternal DNA and shifts by ±FF/2 depending on fetal inheritance:

Fetus inherits maternal ALT

cfDNA VAF ≈ 0.5 + FF/2  (upward shift)

Fetus is REF (did not inherit)

cfDNA VAF ≈ 0.5 − FF/2  (downward shift)

2. Fetal Fraction Estimation

Fetal fraction is estimated from informative SNP sites identified in a panel-wide pileup, computed by fetal_fraction.py. The result is passed as --fetal-fraction to the downstream variant classifier.

2.1 Informative SNP Selection

A site is informative if the maternal genotype is homozygous-alternative (GT=1/1) and the fetus inherits one reference allele from the father. The minor allele in cfDNA (the REF allele from fetal DNA) appears at frequency FF/2.

2.2 Estimation Formula

For each informative site i, the per-SNP FF estimate is:

FF_i = 2 × minor_AF_i where minor_AF = min(alt_AF, 1 − alt_AF) Primary FF = median(FF_i) after IQR-based outlier removal 95% CI: normal approximation ± 1,000-iteration bootstrap (seed 42)

3. Variant Origin Classification

After variant calling with bcftools mpileup on panel target regions, each variant is classified by variant_analysis.py (classify_variant_origin). The two primary inputs are the observed VAF and the estimated FF.

3.1 Tolerance Window

expected_fetal_vaf = FF / 2 tolerance = expected_fetal_vaf × fetal_af_tolerance_factor (default 2.0) = FF At FF=10%: expected_fetal_vaf = 0.05, tolerance = 0.10 Fetal-specific window: VAF ∈ [0, 0.15] (and binomial test must pass)

3.3 Fetal-Specific Binomial Test

A VAF within the tolerance window is necessary but not sufficient for a high-confidence fetal_specific call. A one-sided binomial test is applied against a null error rate of p₀ = 0.001 (sequencing noise floor):

H₀: P(alt) = 0.001 (background sequencing noise) z = (observed_alt_AF − p₀) / sqrt(p₀ × (1 − p₀) / depth) p = 0.5 × erfc(z / sqrt(2)) p < 0.01 → origin = fetal_specific, confidence = high p ≥ 0.01 → origin = possible_fetal, confidence = low

3.4 Maternal Heterozygous: Fetal Inheritance Sub-classification

For maternal_het variants (VAF 0.35–0.65), a secondary test assesses fetal inheritance by comparing the VAF deviation from 0.5 against the expected FF/2 shift:

deviation = VAF − 0.5 |deviation| < tolerance → fetal GT = uncertain (cannot determine) deviation > 0 and |deviation − FF/2| < tolerance → fetal GT = likely_het (fetus inherited ALT) deviation < 0 and |deviation + FF/2| < tolerance → fetal GT = likely_ref (fetus is REF)

5. In Silico cfDNA BAM Simulation

simulate_nipt_bam.py injects synthetic reads at defined loci into a real maternal panel BAM (.target.bam), faithfully reproducing the low-VAF cfDNA signal of fetal-specific variants.

5.1 Spike-in Arithmetic

target_VAF = FF / 2 n_to_add = max(1, round(n_existing × target_VAF / (1 − target_VAF))) Example: depth = 1000, FF = 10% target_VAF = 0.05 n_to_add = round(1000 × 0.05 / 0.95) ≈ 53 synthetic reads

5.2 Read Cloning Algorithm

For each synthetic read to add, the algorithm:

  1. Selects a random template read overlapping the target position.
  2. Maps the reference position to query index via CIGAR alignment pairs (get_aligned_pairs).
  3. Clones the AlignedSegment; renames query_name to {original}_sim_{i}.
  4. Replaces the base at the query position with the spike base (ALT or REF).
  5. Clears the duplicate flag (flag & ~0x400) to prevent MarkDuplicates removal.
  6. Preserves CIGAR, mapping quality, base qualities, and all auxiliary tags.

After all reads are appended, the BAM is sorted and indexed. Original reads are never modified.

6.3 Portal Review Interpretation

Total analyzed: 370 · Maternal het: 211 · Maternal hom: 137 · Fetal: 6

The 211 maternal het and 137 maternal hom variants are real germline variants already present in Test_sgNIPT.target.bam. They were not introduced by the spike-in. The 6 fetal variants represent hotspot spike-in sites that survived variant calling (vc_min_depth=50) and binomial classification at ~5% VAF. Recovery of 6 out of ~21 attempted spikes is expected — not all sites achieve sufficient depth or cross the binomial significance threshold at 5% VAF.

8. Standard Test Sample Generation (make_test_nipt_samples.sh)

The official script builds the FF5/FF10/FF15 test suite using the same simulator with the 6-variant data/test/test_variants.tsv:

for FF in 5 10 15; do simulate_nipt_bam.py \ --bam Test_sgNIPT.target.bam \ --output FF${FF}_sim_Test_sgNIPT.target.bam \ --fetal-fraction 0.${FF} \ --variants data/test/test_variants.tsv \ --ff-vcf Test_sgNIPT.ff_variants.vcf \ --n-ff-snps 100 --seed 42 done

The hotspot and dominant_xlinked samples were created using the same pattern with custom variant TSVs, manually executed via Docker. Both output BAM samplesheet CSVs are consumed by run_sgnipt.sh --input-bam mode, which bypasses FASTQ-to-BAM steps.

9. Pipeline Architecture Overview

The full sgNIPT pipeline (Nextflow v1.2.0, Docker image sgnipt:latest) operates in two modes:

FASTQ Mode (clinical samples)
  1. fastp QC (Q15, min 36 bp)
  2. BWA-MEM2 alignment → GRCh38
  3. Picard MarkDuplicates
  4. samtools target extraction
  5. Fetal fraction estimation
  6. bcftools variant calling
  7. Variant origin classification
  8. VEP annotation
  9. Report generation
BAM Mode (simulation / re-analysis)
  1. — (fastp skipped)
  2. — (BWA skipped)
  3. — (MarkDup skipped)
  4. — (extraction skipped)
  5. Fetal fraction estimation
  6. bcftools variant calling
  7. Variant origin classification
  8. VEP annotation
  9. Report generation
MarkDuplicates and simulated BAMs

Simulated BAMs must be processed in BAM mode (which skips MarkDuplicates) because synthetic reads share the same coordinates as their template reads. Running MarkDuplicates on a spiked BAM would remove cloned reads as optical/PCR duplicates, destroying the intended spike-in signal. This is why simulate_nipt_bam.py uses .target.bam (post-markdup) as input, and the samplesheet triggers BAM-mode entry in the pipeline.
